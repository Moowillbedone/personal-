## 1. 배경 — 원래 요청 (3가지)
1. **회사별 실제 상용 UIUX 화면(스크린샷)** 을 각 케이스에 넣을 것 (그립·캐시워크·하우핏·다노·엔핏 등).
2. **데이터·숫자 기반의 입증/정합화** 작업. (세부 기준·개인 메모는 **로컬 비공개 파일**에 별도 정리해둠 — git에 올리지 않음. ⚠️ 6번 참조)
3. 포트폴리오와 이력서 내용 차이를 **PRD·컨플루언스 자료로 하나씩 싱크** (그립은 재직 중이라 자료 있음, 나머지는 부족할 수 있음).

→ 위 3개는 "콘텐츠" 단계. **이번 작업은 그 앞 단계인 "UI 전면 개선"** 을 먼저 끝낸 것.

## 2. 확정된 방향 (작업 시작 전 3개 질문으로 합의)
| 질문 | 결정 |
|---|---|
| 작업 범위 | **메인 포트폴리오만** 리디자인 (`portfolio-ignite/`는 그대로 둠) |
| 포맷 | **단일 스크롤 웹 + A4 인쇄** (기존 14쪽 A4 덱에서 전환) |

근거: 이그나이트 포트폴리오가 ① 단일 컬러 절제 ② 넉넉한 여백 ③ 명확한 타이포 위계 덕에 "깔끔하고 안 정신없다"고 느꼈고, 그 강점을 메인에 이식.

## 3. 이번에 한 일
### 리디자인
- `portfolio/index.html` **전면 재작성**: 14쪽 A4 덱 → 단일 스크롤. CSS는 인라인(`<style>`)으로 통합, 기존 `portfolio/style.css`는 **삭제**.
- 디자인 시스템(이그나이트 톤): 배경 `#fafafa` / 잉크 `#0a0a0a` / 네이비 `#0a0e1a`·`#1e293b` / 블루 `#1e40af`(+연한 `#dbeafe`) / 골드 `#fbbf24`(+`#fef3c7`,`#92400e`). 폰트 Pretendard + Space Grotesk.
- 회사 구분: 단일 톤 로고칩 `.logo-chip`(네이비 사각형 + 이니셜 G·캐·하·다·엔·언·트·팀).

### 추가한 Pain-point UI
- 상단 **고정 네비 + 읽기 진행바 + 스크롤 스파이**(현재 섹션 하이라이트) + **맨위로 버튼**.
- Hero 직후 **At a glance 밴드**(핵심 지표 6) + 숫자 **카운트업**.
- 스크롤 **페이드업 리빌** — `prefers-reduced-motion`·인쇄 시 자동 OFF.
- 상단 `↓ PDF` 버튼 = `window.print()`, **A4 인쇄 CSS**(주요 섹션·케이스 단위 page-break).

### 변경 파일 맵
| 파일 | 변경 |
|---|---|
| `portfolio/index.html` | 전면 재작성 (단일 파일, 인라인 CSS+JS) |
| `portfolio/style.css` | 삭제 (인라인화) |
| `portfolio/README.md` | 새 구조로 갱신 |
| `portfolio/assets/screens/` | **신규** — 실제 화면 넣는 폴더 + `README.md`(파일명 규칙표) |
| `index.html` (루트 랜딩) | 메인 카드 설명 "14p→9 cases" 갱신 |
| `.gitignore` | 비공개 패턴 `_PRIVATE_*` 추가 |

## 4. 새 포트폴리오 구조 & 동작
섹션 순서(앵커 id): Hero(`#top`) → At a glance → About(`#about`) → Career(`#career`) → Impact(`#impact`) → Cases(`#cases`, 케이스 9건 `#case-1`~`#case-9`) → Skills(`#skills`) → Philosophy(`#philosophy`) → Contact(`#contact`).

케이스 9건(= 회사/서비스): 01 그립 광고플랫폼 · 02 그립 도네이션 · 03 캐시워크 커뮤니티 · 04 언니의파우치 언니딜 · 05 트로스트 EAP · 06 팀워크 미션광고 · 07 하우핏 AI · 08 다노 베이직케어 · 09 엔핏 모두의트레이닝.
각 케이스 구조: **배경 → 접근(A1·A2…) → 구조(Flow 다이어그램) → 화면(Screens) → 결과(Impact) → 학습(Lesson)**.

JS는 바닐라, 라이브러리 없음. 진행바/스파이/리빌/카운트업 전부 IntersectionObserver + rAF.

## 5. ✅ 실제 화면(스크린샷) 넣는 법 — 코드 수정 불필요
1. 이미지를 `portfolio/assets/screens/` 에 **정해진 파일명**으로 저장하면 끝. 빈 자리(점선 박스)가 자동으로 사진으로 바뀜.
2. 파일명 규칙표: **`portfolio/assets/screens/README.md`** 에 케이스별로 정리됨.
   - 예: 그립 광고 → `grip-ad-1.png`(Admin·16:10), `grip-ad-2.png`/`grip-ad-3.png`(앱·9:18) / 언니딜 → `unnie-1~3.png` 등.
3. 비율: 앱=세로(9:18), Admin/대시보드=가로(16:10). 안 맞아도 깨지진 않고 크롭됨.
4. ⚠️ 실데이터 노출 우려 있는 화면은 **마스킹 후** 첨부.

## 6. ⚠️ 비공개 메모 — 집 PC에서 안 보일 수 있음 (중요)
- 데이터 정합/개인 메모용 **비공개 파일이 로컬에만 있음**(`.gitignore`의 `_PRIVATE_*`로 git 제외 → **푸시 안 됨 → 집에서 git clone/pull 해도 안 생김**). 보안상 의도된 동작.
